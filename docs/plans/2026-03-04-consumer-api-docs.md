# Consumer API Documentation

## Overview
- Add OpenAPI spec via swaggo/swag annotations in handler code + `swag init` generation
- Serve Swagger UI at `/api/docs/*` behind consumer auth
- Create quick start guide at `docs/consumer-api.md` (auth, curl examples, link to Swagger UI)
- The hub has 15 consumer-facing API endpoints but no documentation for integrators

## Technical Details

**Path param names must match chi routes exactly:**
- `{id}` — for `/notes/{id}`, `/notes/{id}/archive`, `/tags/{id}`, `/attachments/{id}`
- `{noteID}` — for `/notes/{noteID}/tags`, `/notes/{noteID}/attachments`, `/notes/{noteID}/backlinks`

**ErrorResponse struct** (for `@Failure` annotations):
```go
type ErrorResponse struct {
    Error string `json:"error" example:"descriptive error message"`
}
```

**General API annotations** (above `main()` in `cmd/hub/main.go`):
```
// @title Bear Sync Hub API
// @version 1.0
// @description API for reading and writing Bear notes via the hub.
// @BasePath /
// @securityDefinitions.apikey BearerAuth
// @in header
// @name Authorization
// @description Consumer Bearer token. Format: "Bearer {token}"
```

**Swagger UI mount:**
```go
r.Route("/docs", func(r chi.Router) {
    r.Use(srv.authMiddleware("consumer"))
    r.Get("/*", httpSwagger.Handler(
        httpSwagger.URL("/api/docs/doc.json"),
    ))
})
```

**Swagger generation:**
```bash
swag init -g cmd/hub/main.go --output internal/api/docs --parseInternal
```
`--parseInternal` is required because models are in `internal/models/`.
